# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A single-page marketing site for **BespokeTouch**, a tablet-kiosk product for massage/spa salons (guests mark preferences on a body map; therapists get a briefing before entering). All copy is in **Polish** — keep new copy Polish unless told otherwise. Generated by [v0](https://v0.app) and linked to a v0 project (merges to `main` auto-deploy).

## Commands

```bash
npm run dev      # dev server on :3000 (Turbopack)
npm run build    # production build
npm run start    # serve production build
npm run lint     # eslint — currently NON-FUNCTIONAL: no eslint.config.js exists in the repo
```

Prefer starting the dev server through the Browser preview tools (config in `.claude/launch.json`, name `bt-landing-dev`), not Bash. There is no test suite.

`pnpm-lock.yaml` and `package-lock.json` both exist — the repo's canonical manager is pnpm; npm is used locally. Don't delete either lockfile.

## Architecture

- **Next.js 16 App Router + React 19 + Tailwind CSS v4.** Server Components by default; only interactive pieces (`hero`, `site-nav`, `multilingual`, `reveal`, `kiosk-demo/kiosk-demo`) are `'use client'`.
- **The whole page is `app/page.tsx`** — it imports and orders section components from `components/`. Each section is a self-contained file (`hero.tsx`, `guest-moment.tsx`, `therapist-procedure.tsx`, `silence-ritual.tsx`, `kiosk-demo/kiosk-demo.tsx`, `privacy-section.tsx`, `multilingual.tsx`, `manager-section.tsx`, `closing-cta.tsx`, plus `site-nav.tsx` / `site-footer.tsx`). To change a section, edit its file; to reorder, edit `page.tsx`.
- Import alias `@/*` maps to the project root (e.g. `@/components/...`, `@/lib/utils`).

### The interactive kiosk demo (`components/kiosk-demo/`)

- `kiosk-demo.tsx` is a self-contained stateful simulation: a mocked guest tablet (treatment → body map → preferences → done) on the left and a "live" therapist panel on the right, both driven by the same `useState` in one component — every tap on the left re-renders the right immediately. There's no backend; it's a UI demo only.
- `data.ts` holds local copies of demo content (zones, marker positions, oils, a trimmed treatment catalogue) and `figure-paths.ts` holds the traced body-silhouette SVG path data. `body-map.tsx` has the silhouette/marker/popover/static-map presentational pieces.
- This data and the body-map SVGs are **ported from a separate sibling project**, the real BespokeTouch product app (`bespoke touch/src/...` — a different repo, not part of this one). If the real app's zone list, marker positions, or figure paths change, re-sync manually; there is no shared package.
- Prices were intentionally stripped from this demo's treatment/duration UI — don't reintroduce them unless asked.

### Design tokens (Tailwind v4, CSS-first)

There is **no `tailwind.config.js`**. All theme tokens live in `app/globals.css` under `@theme inline` — the custom color palette (`cream`/`oatmeal`/`sand` neutrals, `charcoal`/`slate` text, `clay`/`rose`/`sage` accents with `-light`/`-dark`/`-tint` variants) and shadows (`shadow-soft`, `shadow-lift`). Use these token classes (`bg-cream`, `text-clay-dark`, etc.) — do not introduce raw hex or off-palette Tailwind colors. Add new tokens in that block.

### Recurring conventions

- **`Reveal` (`components/reveal.tsx`)** wraps section content for a fade-and-rise-on-scroll effect via IntersectionObserver. Pass `delay={ms}` to stagger siblings and `className` for layout (width/margins go here, e.g. `lg:w-3/5`). It honors `prefers-reduced-motion`. The paired CSS (`.reveal` / `.is-visible`) and all keyframe animations are in `globals.css`.
- **Split-line headings:** section `<h2>`/`<h1>` render a regular first line, a `<br/>`, then an `<em>` second line in an accent color (`text-clay-light` on dark bg, `text-clay-dark`/`text-sage-dark` on light bg). Lora's true italic is loaded in `app/layout.tsx` — keep both `style` variants there so `<em>` renders real italic, not synthesized oblique.
- **Section anchors + smooth scroll:** every section has an `id` (`gora`, `gosc`, `terapeuta`, `demo`, `rytual`, `bezpieczenstwo`, `raporty`, `kontakt`), but `components/site-nav.tsx`'s `links` array intentionally only surfaces a subset (currently `gosc`, `terapeuta`, `demo`, `kontakt`) — not every section is meant to be in the nav. Smooth scroll and the fixed-nav offset come from `html { scroll-behavior }` + `[id] { scroll-margin-top }` in `globals.css`. When adding a nav item, give the target section a matching `id`; when removing one from the nav, the section and its `id` can stay.
- **Logo is inline SVG, not an image:** the mark is a path in `lib/logo-path.ts`, rendered by `components/logo-mark.tsx` using `fill-clay-light` / `fill-charcoal` token classes. The favicon is `app/icon.svg` (App Router auto-detects it) built from the same path. Update all three together if the logo changes.
- **Images** live in `public/images/` and are referenced with plain `<img src="/images/...">` (Next image optimization is disabled — see below). Fonts are Plus Jakarta Sans (`--font-sans`) and Lora (`--font-serif`) via `next/font/google` in `app/layout.tsx`.

### Gotchas

- `next.config.mjs` sets `typescript.ignoreBuildErrors: true` and `images.unoptimized: true` — **`npm run build` will NOT catch type errors**. Run `npx tsc --noEmit` for real type checking.
